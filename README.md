# 📊 Superstore Sales Forecasting & Operations Portal

A complete, end-to-end data science and machine learning system built to solve multi-layered time-series forecasting, anomaly detection, product clustering, and inventory stocking problems for a mid-to-large scale e-commerce and retail enterprise.

---

## 🚀 Interactive Portal Preview
The system includes a fully functional interactive Streamlit dashboard consisting of:
1. **Sales Overview Dashboard**: Historical trends, sales metrics, and interactive filters for categories and regions.
2. **Forecast Explorer**: Dynamic 3-month out-of-sample sales projections generated by an autoregressive XGBoost model with MAE and RMSE metrics.
3. **Anomaly Report**: Side-by-side weekly anomaly monitoring using **Isolation Forest** (macro-level) and **Rolling Z-Score** (local variations).
4. **Product Demand Segments**: PCA visualization of demand groups and actionable stocking guidelines (JIT vs. safety buffers).

## ⚙️ Methodology & Architecture

### 1. Data Exploration & Time Series Decomposition
*   Datetime parsing and feature extraction (Year, Month, Week Number, Day of Week, Quarter, Season).
*   Identified **Technology** as the highest revenue driver ($836,154.03) and the **East** as the most consistent growth region.
*   Conducted time series decomposition (Trend, Seasonality, Residuals) and checked for stationarity using the **Augmented Dickey-Fuller (ADF) Test**. Differencing was applied to achieve stationarity.

### 2. Forecasting Model Comparison
We compared three different forecasting approaches over a 3-month validation horizon:
*   **SARIMA**: Order $(0,1,2)\times(0,1,1)_{12}$ statistical model.
*   **Facebook Prophet**: Additive regression model with weekly/yearly seasonality.
*   **XGBoost Regressor**: Supervised ML using lag features (`lag1`, `lag2`, `lag3`), moving averages, and seasonal indicators.

**Performance Summary Table:**
| Model | MAE ($) | RMSE ($) | MAPE (%) | Month 1 Forecast | Month 2 Forecast | Month 3 Forecast |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **XGBoost (Best)** | **4,463.78** | **5,373.11** | **10.66%** | **$26,022.51** | **$24,787.58** | **$35,064.25** |
| Facebook Prophet | 6,569.11 | 7,654.20 | 17.50% | $29,512.44 | $25,123.88 | $31,902.12 |
| SARIMA | 6,231.58 | 7,213.26 | 24.51% | $35,123.37 | $21,114.98 | $28,912.44 |

*XGBoost was recommended for production use due to its superior error metrics and robust autoregressive forecast accuracy.*

### 3. Anomaly Detection
*   **Isolation Forest (Global)**: Used to isolate macro-level operational spikes or drops.
*   **Rolling Z-Score (Local)**: Flags weekly sales deviating $>2$ standard deviations from an 8-week local window mean.
*   Correlated anomalies to holiday surges (Thanksgiving/Black Friday) and off-season procurement pauses.

### 4. Product Clustering (K-Means)
Segmented 14 sub-categories based on total sales volume, growth rate, volatility, and Average Order Value:
*   **Cluster 0 (Stable Low-Volume Commodities)**: appliances, art, paper $\rightarrow$ *Strategy*: **Just-In-Time (JIT)**.
*   **Cluster 1 (Premium High-Volume Drivers)**: chairs, copiers, phones $\rightarrow$ *Strategy*: **Dynamic Safety Buffers**.
*   **Cluster 2 (Hyper-Growth Outliers)**: binders $\rightarrow$ *Strategy*: **Agile Promotional Scaling**.

## 🛠️ How to Setup and Run Locally

### 1. Clone the repository
```bash
git clone https://github.com/shreyakargutkar/SalesForecasting-System.git
cd SalesForecasting-System
```

### 2. Install dependencies
Ensure you are using Python 3.x and run:
```bash
pip install -r requirements.txt
```

### 3. Start the Streamlit Dashboard
```bash
streamlit run app.py
```
Open **`http://localhost:8501`** (or the port specified in your terminal) to explore the system.
